Service :
/hint_loader_service /armor_interface_srv /oracle_solution

This node listens for service requests from the 'hint_collector' node via the 'hint_loader_service'. Upon receiving a request, it loads the hint into the ARMOR reasoner, activates the reasoner to deduce a hypothesis based on the previously loaded hints, and queries the ARMOR reasoner for a list of 'COMPLETE' hypotheses. If the newly deduced hypothesis is marked 'COMPLETE', it then verifies its 'CORRECTNESS'.

def hint_loader.clbk_oracle_service (   msg)

This is the 'clbk_oracle_service' function for the oracle node.

Returns
Bool

This function serves as a callback for the '/oracle_service' node. It waits for a service request from the 'motion_controller' node. Depending on the type of request received, it loads the hint into the ARMOR reasoner, starts the reasoner to deduce a hypothesis based on the previously loaded hints, and queries the ARMOR reasoner for a list of 'COMPLETE' hypotheses. If the hypothesis is 'CONSISTENT', it returns 'True'; otherwise, it returns 'False'. If the hypothesis is 'CONSISTENT', the user can also request the node to check if the hypothesis is 'CORRECT'. A hypothesis is considered correct if it is 'CONSISTENT' and belongs to a predefined list of correct hypotheses within this node. If the hypothesis is correct, the node returns 'True'; otherwise, it returns 'False'.
